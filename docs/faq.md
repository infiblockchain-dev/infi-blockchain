# INFI FAQ

## Is INFI live on mainnet?

No. INFI is currently an early prototype.

## What is the gas token?

Invertx.

## What is test InvertX?

test InvertX is the INFI Testnet gas token. It is free through the testnet faucet, non-tradable, and has no redeemable real-world value.

## Why display 1 test InvertX = 1.25 USDT on testnet?

Wallets and INFI Scan may show that as the InvertX reference value for testnet UI. test InvertX itself is non-tradable and is not a market price, redemption value, or financial promise.

## Is Invertx an ERC-20?

Native Invertx is not an ERC-20. It is the native gas token, like ETH on Ethereum. A wrapped ERC-20 version can be added later.

## Will INFI support MetaMask?

That is the goal. INFI should support EVM wallets through Ethereum JSON-RPC and standard wallet import methods.

## Will INFI support WalletConnect?

Yes. INFI dapps should support WalletConnect QR code and mobile deep-link flows.

## Will INFI support non-EVM chains?

The plan is EVM-first, then non-EVM support through adapters for ecosystems such as Solana, Bitcoin, Cosmos, and others.

## Can INFI be impossible to hack?

No system can honestly promise that. INFI should focus on reducing risk through audits, testing, simple design, transparency, bug bounties, and decentralization.

## What is INFI Scan?

INFI Scan is the planned explorer website where users can search transactions, blocks, addresses, validators, contracts, and later cross-chain activity.

## What is SbSe Protocol?

SbSe Protocol is planned as a future governance logic engine node for INFI.

## Will INFI support tokenized assets?

Yes. INFI should support decentralized tokenized assets with asset registries, issuer transparency, metadata verification, permission visibility, and INFI Scan risk labels.

## Will INFI have free domains?

Yes. INFI should include free on-chain domains such as `name.infi`. Normal public browser access to `.infi` names requires a DNS, gateway, browser extension, wallet browser, or resolver strategy.

## Will INFI include a VPN function?

The plan includes a future decentralized VPN function using separate provider nodes, on-chain registry/payment logic, reputation, and privacy-focused user experience. It should not be part of validator consensus.

## Can I deploy Solidity contracts now?

Not yet. The current prototype does not include real EVM execution. That is a core upcoming milestone.
